# Flowcast Landing Page Design

## 1. 页面目标

Landing Page 只承担一个任务：建立用户对 Flowcast 的正确心智。

> 流转 Flowcast 是跨模态的专属内容知识库与流转引擎。

用户进入页面后，需要快速理解：

- Flowcast 不是普通 AI 写作工具，也不是单平台文案生成器。
- Flowcast 的核心价值是把已有内容沉淀为创作者画像，再流转成多个平台可发布的内容版本。
- 当前可用主链路是视频内容进入后，完成转写、解析、画像确认和多平台内容输出。
- 音频、图文、文本、视频脚本、短视频拆条等形态属于产品版图，但需要明确标注开发状态。
- Landing Page 不锚定长视频，短视频、口播、图文和文章都可以是内容源。短视频和口播更容易集中观点、提炼闪光点，也更适合展示“同一内容流转到多个平台”的效果。

## 1.1 Landing 文案规则

Landing Page 上所有可见文案都不允许使用句号。

执行规则：

- 中文文案不使用 `。`
- 英文文案不使用句末 `.`
- 标题、按钮、标签、卡片说明、FAQ 答案都遵守这条规则
- 如需停顿，优先使用换行、顿号、逗号、冒号或更短的短句
- 文档说明可以正常使用标点，但所有进入页面的展示文案必须去掉句号

## 2. 首屏信息

### 2.0 气泡（放在主标题上面，就和概念图里的一样）
跨模态的专属内容知识库与流转引擎

### 2.1 主标题

```text
让视频、音频、图文、文本自由流转
```

### 2.2 副标题

```text
把已有内容沉淀成创作者画像，再根据你的观点、语气和结构，流转成适合不同平台发布的内容版本
```

### 2.3 CTA

首屏只保留两个主入口：

- `创建创作者画像`
- `进入流转工作台`

CTA 对应关系：

- `创建创作者画像` 进入资产库 / 创作者画像创建流程。
- `进入流转工作台` 进入流转工作台。

不使用 `创建声纹资产库`、`拆解一条内容` 等过时或不准确文案。

### 2.4 导航

顶部导航建议：

- 流转路径
- 平台输出
- 创作者画像
- 多形态版图
- 常见问题

不放定价入口。当前 MVP 暂不展示付费定价结构。

## 3. 首屏核心视觉

首屏不再使用五张等权功能卡。

旧方案的问题是每个步骤都像一张独立功能卡，视觉上更接近“功能说明”，没有形成内容正在流动、汇聚、分发的感觉。

新版首屏核心视觉要做成“内容流转线路图”：

```text
多形态输入
  视频 / 音频 / 图文 / 文本
        ↓
Flowcast 引擎
  转写解析 / 观点提取 / 画像匹配 / 写入确认
        ↓
多平台输出
  小红书 / 抖音 / B 站 / 知乎 / X
```

设计重点：

- 多个输入形态从左侧或上方汇入 Flowcast 引擎。
- Flowcast 引擎作为中枢，而不是普通步骤卡。
- 输出平台向右侧或下方分流，形成明确扩散感。
- 当前可用的视频链路高亮。
- 音频、图文、文本不隐藏，但标注 `开发中`。
- 用户确认写入画像是关键节点，需要在视觉上明确出现。

## 4. 平台输出区

### 4.1 设计意图

这一节用一个固定内容样本，展示同一内容如何被流转成不同平台版本。

源内容示例：

```text
一条 60 秒口播短视频：创作者讲清自己对 AI 内容复用的一个核心观点
```

页面展示它被改写为不同平台的内容样张。重点不是只展示平台 Logo，而是让用户看到同一份内容在不同平台中的表达差异。

示例优先使用短视频、口播或一段图文观点，不使用长视频作为默认样本。长视频、直播回放、课程录屏可以支持，但不应成为 Landing Page 的默认叙事锚点。

### 4.2 必须覆盖的平台

每个平台必须使用正确 Logo：

- 小红书
- 抖音
- B 站
- 知乎
- X

实现前需要先盘点 `public/logos` 中现有 Logo 资产。不完整或不准确的 Logo 需要补齐或替换，不允许用错误 Logo 或纯文本占位冒充。

### 4.3 平台样张内容方向

- 小红书：短段落、强场景、标题直接。
- 抖音：30 秒口播脚本，开头抓人。
- B 站：动态 / 简介，结构完整。
- 知乎：结论先行、论证感强。
- X：短句观点串。

这一节可以用横向滚动、平台矩阵或分组样张。关键是每个平台都应该有自己的表达形态，而不是只替换平台名。

## 5. 创作者画像区

创作者画像只展示 5 类核心内容：

- 定位
- 观点
- 语气
- 常用结构
- 禁用表达

这一节要说明：

- Flowcast 的输出不是泛 AI 文案。
- 系统会调用创作者画像，让内容尽量保留创作者本人的观点、语气和结构。
- 上传或解析内容后，系统只生成画像变化建议。
- 用户确认后，变化才会写入创作者画像。

建议视觉结构：

```text
源内容解析结果
        ↓
画像变化建议
        ↓
用户确认写入
        ↓
后续生成更像本人
```

这里的重点是“可控沉淀”，不是夸大自动学习能力。

## 6. 多形态流转版图

多形态流转版图不隐藏。它应该成为页面里第二个强记忆点。

这一节要直接展示 Flowcast 的产品方向：

```text
输入形态
  视频 / 音频 / 图文 / 文本

中间层
  内容知识库 / 创作者画像 / 平台表达规则

输出形态
  多平台文字稿 / 视频脚本 / 图文草稿 / 短视频拆条 / 文生视频
```

状态标记：

- 视频：可用。
- 多平台文字稿：可用或部分可用。
- 音频：开发中。
- 图文：开发中。
- 文本：开发中。
- 视频脚本：开发中。
- 短视频拆条：开发中。
- 文生视频：暂不承诺，解释为能力尚未成熟。

设计要求：

- 不要把远期形态藏到 FAQ 里。
- 不要暗示尚未完成的能力已经完整可用。
- 用状态标签让用户同时看到产品愿景和当前边界。
- 视觉上继续强化“输入汇入中间层，再流向输出形态”的流转感。

## 7. FAQ

FAQ 需要解释边界和开发中能力，不写泛泛的“敬请期待”。

### 7.1 为什么当前先开放视频转文字？

视频是创作者最常见的表达入口，尤其是短视频和口播，观点更集中、闪光点更清楚，转写解析后更适合快速流转成小红书、抖音、B 站、知乎和 X 等平台版本。长视频、直播回放和课程录屏也可以支持，但不是 Landing Page 的默认样本。

### 7.2 为什么暂时不支持文转视频？

文生视频能力仍在快速发展中，稳定性、可控性和成片质量还不足以支撑严肃创作者的日常发布。Flowcast 会先把内容理解、画像沉淀和文字流转链路做好。

### 7.3 音频、图文、文本什么时候支持？

这些形态都在 Flowcast 的产品版图内，当前还在开发中。页面可以展示入口和方向，但不能把它们伪装成已经完整可用。

### 7.4 上传内容会自动写入创作者画像吗？

不会。系统只会提出画像变化建议，用户确认后才会写入创作者画像。

### 7.5 是否会自动发布到平台？

暂不支持自动发布。当前重点是生成可编辑、可复制、可沉淀的多平台内容版本。

## 8. 页面顺序

最终页面建议顺序：

1. Hero：定位、主标题、副标题、两个 CTA、流转线路图。
2. 平台输出：同一内容流转成不同平台版本，并使用正确 Logo。
3. 创作者画像：定位、观点、语气、常用结构、禁用表达。
4. 多形态流转版图：输入、中间层、输出和状态标签。
5. FAQ：解释当前边界和开发中能力。
6. 底部 CTA：`创建创作者画像` / `进入流转工作台`。

## 9. 视觉方向

延续现有品牌基础：

- 温白纸面背景。
- 墨色中文文字。
- 朱砂色 `#b4472f` 作为关键操作和当前节点强调色。

但不要继续做成复古档案室。新版 Landing 应该更像“内容流转控制台”：

- 横向流线。
- 多输入汇聚。
- 中枢引擎。
- 平台矩阵。
- 输出样张分流。
- 清晰状态标签。

避免：

- 紫蓝 SaaS 渐变。
- 抽象 AI 光效。
- 过度复古纹理。
- 五张等权功能卡。
- 只展示三四个平台。
- 用错误平台 Logo。
- 暗示自动发布、完整视频生成或完整文生视频能力。

## 10. 验收标准

- 首屏副标题必须使用“跨模态的专属内容知识库与流转引擎”方向。
- 首屏 CTA 必须是 `创建创作者画像` 和 `进入流转工作台`。
- Landing Page 所有可见文案不得使用中文句号 `。` 或英文句末 `.`
- 首屏视觉必须呈现内容从输入形态汇入 Flowcast，再分流到多个平台。
- 平台输出区必须覆盖小红书、抖音、B 站、知乎、X。
- 每个平台必须使用正确 Logo。
- 多平台输出必须展示同一内容在不同平台的表达差异。
- 创作者画像只展示定位、观点、语气、常用结构、禁用表达五类核心内容。
- 多形态流转版图必须直接展示，不隐藏。
- 所有尚未支持的形态必须明确标注开发中或暂不承诺。
- FAQ 必须解释暂不支持文转视频和其它形态的原因。
- 页面不展示定价、自动发布承诺、历史任务列表或复杂工作台状态。
- Landing Page 不以长视频作为默认叙事样本，优先使用短视频、口播或一段图文观点展示流转效果。
