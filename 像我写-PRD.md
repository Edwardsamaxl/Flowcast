# 像我写 - 产品需求文档

## 1. 产品概述

**像我写** 是面向中文知识创作者的「个人声纹资产库 + 视频内容转化引擎」。它通过学习创作者过往视频、口播和直播内容，建立个人声纹资产库，帮助创作者把长视频、直播回放、课程录屏等口播内容，转化成像本人写的小红书、知乎和 X（Twitter）文字内容。

核心价值不是「AI 帮你写内容」，而是：

> 让 AI 学会你的观点、语气和表达方式，把你的视频内容转化成像你本人会写的平台文字内容。

## 2. 背景与机会

现有工具已经覆盖了很多基础能力：

- 轻析等工具已经能做视频解析、转写、小红书笔记、口播稿、知识笔记和思维导图。
- NotebookLM 已经擅长资料理解、知识整理、问答和笔记生成。
- Jasper、Writer、Copy.ai、Typeface 等海外产品已经证明「品牌语气 + 知识库 + 内容生成」是成熟方向。
- ChatGPT Projects、GPTs、Claude Projects 也能通过文件和指令做低配版的个人知识库写作。

因此，「像我写」不应该做通用视频解析，也不应该做通用 AI 写作。它要切入一个更窄的问题：

**中文知识创作者不是缺一个会写的 AI，而是缺一个真正理解自己声纹资产的 AI 转化助理。**

知识创作者的真实痛点：

- 通用 AI 能写，但写出来不像本人。
- AI 生成内容容易有模板感、营销腔和平台套路感。
- 个人观点、反复强调的方法论、案例散落在历史视频内容里，难以复用。
- 同一个观点在小红书、知乎、X 里需要不同表达方式。
- 创作者越成熟，越在意「这是不是我会说的话」。
- 已经有了大量视频/口播内容，但拆成文字平台内容耗时且容易失真。

## 3. 产品定位

### 3.1 一句话定位

**像我写：中文知识创作者的个人声纹资产库，让 AI 写得像你。**

### 3.2 用户心智

用户打开「像我写」不是为了得到一篇泛泛的 AI 文案，而是为了：

- 把长视频/直播/课程录屏拆成小红书、知乎、X 可发布的文字内容。
- 让 AI 学会自己的表达方式，转化结果像本人写的。
- 减少跨平台改写的时间和失真。
- 沉淀个人声纹资产，越用越像自己。

### 3.3 不做什么

- 不做通用视频解析平台。
- 不做 NotebookLM 式资料问答和知识笔记。
- 不做思维导图。
- 不做排期日历。
- 不做多平台自动发布。
- 不做泛泛的爆款标题生成器。
- 不做面向大型企业品牌治理的重型内容中台。
- 不做朋友圈内容生成（朋友圈为真人日常表达，不适合 AI 代写）。
- 不做公众号内容生成（不在目标平台范围内）。
- 不做口播稿生成（视频/口播是输入来源，而非输出目标）。

## 4. 目标用户

### 4.1 核心 ICP

第一阶段只服务中文知识创作者，优先选择已经有内容资产和变现路径的人。

| 用户类型 | 特征 | 核心痛点 | 付费意愿 |
|---|---|---|---|
| 考研/考证/语言类博主 | 有课程、资料包、长视频或直播课 | 干货多，但拆成平台内容耗时且容易失真 | 高 |
| 职场/成长类知识博主 | 持续输出方法论、案例和观点 | 想高频输出，但 AI 写不出本人判断力 | 中高 |
| 咨询/教练型个人 IP | 内容用于获客，转化到咨询/私域 | 需要既专业又像真人，不想硬广 | 高 |
| 知识付费小团队 | 1-3 人运营，创始人是核心 IP | 助理能执行，但写不出创始人口吻 | 高 |

### 4.2 暂不服务

- 娱乐、剧情、颜值、搞笑类账号。
- 纯电商带货账号。
- 没有历史内容的新手创作者。
- 只想做资料总结、思维导图、课堂笔记的人。
- 只需要低价批量洗稿的人。

## 5. 竞品与差异化

### 5.1 竞品分层

| 类型 | 代表产品 | 已有能力 | 像我写的避让点 |
|---|---|---|---|
| 视频解析/二创工具 | 轻析、BibiGPT 等 | 视频转写、总结、小红书笔记、口播稿 | 不把解析作为主战场 |
| 知识整理工具 | NotebookLM | 资料理解、问答、笔记、音频概览 | 不做通用知识笔记 |
| 品牌 AI 写作平台 | Jasper、Writer、Copy.ai、Typeface | Brand Voice、知识库、营销内容生成 | 不做企业级品牌中台 |
| 通用 AI 工作区 | ChatGPT Projects、Claude Projects | 文件、指令、项目知识、写作风格 | 用垂类工作流和资产结构降低使用门槛 |

### 5.2 核心差异

| 维度 | 通用竞品 | 像我写 |
|---|---|---|
| 服务对象 | 企业品牌、营销团队、泛创作者 | 中文知识创作者和个人 IP |
| 核心资产 | 品牌语气、产品资料、营销知识库 | 观点库、表达风格库、案例库、转化钩子库 |
| 输出目标 | 广告、博客、邮件、社媒帖 | 小红书、公众号、知乎、朋友圈、口播稿 |
| 价值主张 | 提升内容生产效率 | 降低改稿成本，保持人设一致，复用个人声纹资产 |
| 差异壁垒 | 平台能力、团队协作 | 私有声纹资产、垂类模板、改稿学习闭环 |

## 6. 核心假设

产品成立必须证明以下假设：

1. 知识创作者愿意沉淀历史内容、观点和转化素材，建立个人声纹资产库。
2. 用户愿意为「更像我、少改稿、能转化」付费，而不只是为「能生成」付费。
3. 通过历史内容和用户反馈，可以明显降低 AI 味和模板感。
4. 用户愿意为「把视频内容转化为像本人写的平台文字」付费，而不只是为「能生成」付费。
5. 小红书、知乎、X 是中文知识创作者日常内容分发的核心文字平台。
6. 「不像我检测」和「改稿学习」能形成持续使用理由，而不是一次性工具。

## 7. 核心功能

### 7.1 个人声纹资产库

用户建立一个自己的声纹资产库，而不只是上传一堆资料。

资产类型包括：

- 历史视频内容：过往视频、直播回放、课程录屏的转录文本。
- 观点资产：高频观点、方法论、判断标准、反对什么、坚持什么。
- 案例资产：个人故事、学生案例、客户案例、失败案例、对比案例。
- 转化资产：课程介绍、资料包、咨询服务、社群介绍、私域引导方式。
- 表达禁区：禁用词、不喜欢的标题风格、不接受的销售方式。

### 7.2 风格画像

系统自动分析并生成「个人表达画像」：

- 人设定位：如考研陪伴型学姐、职场教练、专业顾问、犀利老师。
- 语气特征：克制、直接、温和、锋利、故事化、专业型。
- 句式偏好：短句、反问、结论先行、案例开头、层层递进。
- 标题偏好：问题型、反常识型、经验型、避坑型、方法型。
- 常用结构：误区 -> 原因 -> 方法 -> 行动建议。
- 常用 CTA：评论领取、私信咨询、加入社群、看课程、收藏转发。
- 禁用表达：不符合本人风格的词、过度营销话术、平台套路句。

### 7.3 内容创作引擎

MVP 阶段聚焦一种核心创作模式：

| 模式 | 输入 | 输出 |
|---|---|---|
| 从长内容拆条 | 视频转录、直播稿、课程录屏文本 | 小红书、知乎、X 可发布内容 |

每次生成都调用：

- 个人声纹画像。
- 观点资产库。
- 平台写作规则。
- 禁用表达。

### 7.4 平台写法引擎

同一个视频内容可以被转化成不同平台版本：

- 小红书：强场景、短段落、痛点标题（≤20 字）、轻转化。
- 知乎：结论先行、可信度、专业判断、弱化营销感。
- X (Twitter)：开头抓人、短句为主、口语化、适合观点输出和轻互动。

### 7.5 不像我检测（MVP 轻量版）

生成内容后，系统提供一段文字分析（由 LLM 判断），指出可疑句子：

- 这句话不像你的常用表达。
- 这段过度营销。
- 这个标题太像模板。
- 这里没有调用你的观点资产。
- 这里可以引用你过去说过的案例。

MVP 阶段不做四维度量化打分，以文字分析为主。

### 7.6 改稿学习闭环（MVP 轻量版）

用户编辑内容后，系统提供反馈标签：

- 太 AI 味了
- 不像我
- 太长了
- 太营销了
- 其他（手动输入）

用户点击后，系统将对应偏好增量记录进声纹资产库，下一次生成时作为负向约束写入 prompt。

MVP 阶段不做自动 diff 分析和自动更新资产库，以用户明确反馈为主。

### 7.7 内容对比

MVP 不做实时对比（单次生成只输出声纹版）。

首页保留固定示例对比（后续使用用户自己的视频作为案例），并添加反馈按钮收集用户对差异明显度的意见。

目的不是炫技，而是让用户清楚看到「声纹资产库」带来的差异。

## 8. MVP 范围

### 8.1 必做

- 本地上传视频，异步语音转文字（本地 faster-whisper）。
- 自动生成个人声纹画像（观点、语气、禁用词等）。
- 手动编辑声纹画像、禁用词、观点列表。
- 只做「从长内容拆条」一种创作模式。
- 生成小红书、知乎、X 三类内容。
- 支持用户编辑生成结果。
- 支持「太 AI 味了」「不像我」「太长了」「太营销了」等反馈标签。
- 保存用户反馈，增量更新声纹偏好。

### 8.2 可先手工完成

- 声纹画像用 prompt + JSON 生成，不训练模型。
- 改稿学习先记录规则和偏好，不做真正微调。
- 观点资产先做结构化列表，不做复杂向量知识库。
- 视频链接解析后续调研开源方案，MVP 阶段先做本地上传。

### 8.3 暂不做

- 视频链接解析（MVP 后调研开源方案）。
- 自动发布。
- 排期日历。
- 思维导图。
- NotebookLM 式问答。
- 团队协作。
- 多账号管理。
- 浏览器插件。
- 用户管理后台、支付套餐系统。
- 实时「通用 AI 版本 vs 像我写版本」对比。
- 四维度量化打分（像我程度 / AI 味 / 平台适配度 / 转化自然度）。

## 9. 用户流程

```text
创建声纹资产库
  -> 上传视频（本地文件）
  -> 系统异步语音转文字
  -> 系统自动提取观点、语气、禁用词等生成声纹画像
  -> 用户确认并修改声纹画像

创作新内容
  -> 选择已转录的视频
  -> 选择输出平台（小红书/知乎/X）
  -> 系统调用声纹资产库生成内容
  -> 用户编辑并反馈
  -> 系统增量更新声纹资产库
```

## 10. 关键页面

### 10.1 首页

主文案：

> 让 AI 写得像你。

副文案：

> 建立你的个人声纹资产库，让 AI 学会你的观点、语气和表达方式。无论是长视频还是直播内容，都能转化成像本人写的平台文字内容。

核心 CTA：

- 创建我的声纹资产库
- 试写一篇内容

### 10.2 声纹资产库页

模块：

- 视频列表（历史转录内容，即案例库）。
- 观点库。
- 声纹画像（人设、语气、句式、标题偏好、常用结构、禁用表达）。
- 平台写法规则。
- 支持手动编辑所有字段。

### 10.3 内容创作页

模块：

- 选择已转录的视频。
- 选择输出平台：小红书、知乎、X。
- 生成内容。

### 10.4 编辑与校准页

模块（两栏布局）：

- 左侧：视频信息 + 转录原文。
- 右侧：平台选择 + 生成结果编辑区 + 操作按钮（重新生成、保存）。
- 底部：反馈标签（太 AI 味 / 不像我 / 太长了 / 太营销了 / 其他）。
- 声纹画像与 AI 沟通按需唤出（标签页 / 滑出面板 / 浮动球，具体形式待进一步决策）。

## 11. 输出内容规范

### 11.1 小红书

- 标题不超过 20 字。
- 强场景、强痛点，但避免夸张标题党。
- 正文短段落，适合手机阅读。
- 保留创作者本人常用表达。
- CTA 自然，不硬塞。
- Emoji 适度，不强求。

### 11.2 知乎

- 先给结论。
- 强调专业判断、经验和证据。
- 避免过强营销感。
- 适合长尾搜索和专业形象建立。
- 段落可较长，逻辑完整。

### 11.3 X (Twitter)

- 开头抓人（前 50 字定生死）。
- 短句为主，可接受口语化甚至轻微不完整句式。
- 适合观点输出和轻互动。
- 可加 1-2 个 hashtag。
- 避免长段落堆砌。

（注：平台规则先用标准模板，后续根据实际使用细化。）

## 12. 技术方案

### 12.1 前端

- Next.js 14
- Tailwind CSS
- 文本编辑器
- 声纹资产管理界面
- 视频上传与异步状态展示

### 12.2 后端

- Node.js
- SQLite
- DeepSeek LLM API
- 本地 faster-whisper（CTranslate2 优化版）用于语音识别
- 后续可调研开源视频链接解析方案

### 12.3 核心数据对象（MVP 精简版）

| 表名 | 作用 |
|---|---|
| User | 用户基础信息 |
| CreatorProfile | 创作者基本信息（昵称、领域等） |
| VoiceProfile | 声纹画像（JSON 存储：语气、禁用词、观点列表、平台规则等） |
| VideoTranscript | 视频转录文本（原始文本 + 提取的观点/特征缓存） |
| GeneratedDraft | 生成的草稿（输入、输出平台、生成结果、用户编辑后的结果、反馈标签） |

案例库由 VideoTranscript 的集合构成。改稿反馈直接增量写入 VoiceProfile 的 JSON 中。

```json

  "beliefs": [
    "学习计划必须能落地，否则再完美也没用",
```

## 13. 验证计划

### 13.1 验证目标

MVP 为演示级 Web 产品，优先用于向评委/投资人展示并小范围验证。

目标：

- 使用自己的自媒体视频作为核心案例。
- 上传 3 篇以上历史视频，生成稳定的声纹画像。
- 测试「从长内容拆条」到小红书、知乎、X。
- 观察生成内容的「像本人程度」和修改成本。
- 小范围邀请 3-5 位目标用户试用并收集反馈。

### 13.2 成功指标（MVP 阶段）

| 指标 | 目标 |
|---|---:|
| 视频转录准确率可用 | 可接受水平 |
| 生成内容比通用 AI 明显更像本人 | 核心评判标准 |
| 单条内容修改时间 | 降到 10 分钟以内 |
| 小范围测试用户认可价值 | 3/5 人 |

### 13.3 失败信号

- 生成内容仍然有明显 AI 味和模板感。
- 视频转录质量太差，导致后续生成不可用。
- 用户改稿时间没有明显下降。
- 声纹画像提取不准，用户手动修正成本过高。
- 输出平台适配度差（不像小红书/知乎/X 的本土风格）。

## 14. 商业模式

### 14.1 商业模式（MVP 后规划）

MVP 阶段不做付费和套餐系统，优先验证产品价值。验证成功后考虑：

| 套餐 | 价格建议 | 功能 |
|---|---:|---|
| 体验版 | 0 元 | 1 个声纹资产库，3 次生成 |
| 创作者版 | 99-199 元/月 | 1 个声纹资产库，每月 30 次生成 |
| IP 团队版 | 499-999 元/月 | 多声纹资产库，团队协作 |

（具体定价和商业模式待 MVP 验证后根据用户反馈确定。）

## 15. 风险与应对

| 风险 | 严重度 | 应对 |
|---|---|---|
| 与 ChatGPT Projects 差异不明显 | 高 | 用垂类声纹资产结构、平台模板和不像我检测降低使用门槛 |
| 风格模仿不够像 | 高 | 先聚焦单一垂类，人工校准声纹画像 |
| 用户不愿上传视频 | 中 | 降低上传门槛，展示画像变化的价值 |
| 输出仍需大量修改 | 高 | 把改稿反馈作为核心闭环，不把一次生成当终点 |
| 被轻析快速复制 | 中 | 不打解析能力，打私有声纹资产和长期学习 |
| 产品过宽 | 高 | MVP 只做小红书、知乎、X，不做全平台 |

## 16. 推荐 Demo 路径

1. 准备 3 篇以上自己的自媒体视频。
2. 上传视频，系统异步转录并提取声纹画像。
3. 系统展示人设、观点、禁用词、平台写法。
4. 用户确认并修改声纹画像。
5. 选择一段已转录的视频，生成小红书版本。
6. 用户编辑并选择反馈标签（如「少点 AI 味」）。
7. 系统增量更新声纹画像并重新生成。
8. 同一视频再生成知乎版本和 X 版本，展示平台适配差异。
9. 展示固定示例对比（用自己的视频作为案例）。

## 17. 产品口号

候选：

- 让 AI 写得像你。
- 训练你的个人声纹资产库。
- 不是 AI 代写，是 AI 学会你的写法。
- 把你的观点、语气和转化方式变成可复用资产。

推荐主口号：

> 让 AI 写得像你。

推荐副口号：

> 建立你的个人声纹资产库，让 AI 学会你的观点、语气和表达方式。无论是长视频还是直播内容，都能转化成像本人写的平台文字内容。
