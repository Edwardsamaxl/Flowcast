# 像我写：前端设计文档

## 0. Direction Lock

**风格方向：出版编辑室**

像我写不是通用 AI 写作器，也不是冷白色的 Notion 仿品。它应该像一个现代中文编辑部工作台：用户把长视频、直播、课程里的口播内容沉淀成自己的表达档案，再从档案里生成可校准、可编辑、可发布的稿件。

界面关键词：

- **编辑感**：像校样、批注、稿纸、档案，而不是营销 SaaS。
- **中文质感**：标题有出版物的力度，正文长时间阅读不累。
- **可信任**：强调来源、规则、修改痕迹和反馈闭环。
- **有温度的克制**：不做手账、不做国风、不做复古纸纹，但避免冷灰蓝带来的疏离感。

一句话判断：用户打开后应该觉得“这是我的中文表达编辑室”，不是“又一个 AI 生成按钮”。

## 1. Visual Thesis

当前版本的问题不是不够干净，而是干净到失去性格：

- 冷白、灰蓝、细阴影让界面发灰。
- 楷体点缀和现代 UI 黑体割裂，容易显得不稳定。
- 大量浅色卡片让所有内容权重相近，缺少主稿件和批注层级。
- 文档和源码出现中文乱码，导致设计和实现都不可维护。

新版改成出版编辑室：

- 背景使用温白和淡墨，不使用冷蓝灰。
- 主色用克制朱砂红，只承担当前状态、重点批注和主操作。
- 标题使用中文衬线气质，UI 和正文保持清晰黑体。
- 页面从“卡片拼贴”改为“稿件区 + 档案区 + 批注区”的工作结构。

## 2. Color Tokens

视觉比例：78% 温白纸面，17% 墨色文字和线条，5% 朱砂强调。

| Token | Value | Role |
|---|---:|---|
| `paper-0` | `#fffdf8` | 页面主背景 |
| `paper-50` | `#f8f3ea` | 次级纸面、侧栏 |
| `paper-100` | `#efe7d9` | 分区底色、表头 |
| `paper-200` | `#ded2bf` | 强分隔线 |
| `ink-950` | `#201b17` | 主标题、正文强文字 |
| `ink-800` | `#3f352d` | 正文 |
| `ink-600` | `#75685a` | 辅助说明 |
| `ink-400` | `#a79a8a` | 占位、弱标签 |
| `seal-50` | `#f8e8df` | 选中底色、轻提示 |
| `seal-500` | `#b4472f` | 主操作、批注、当前状态 |
| `seal-600` | `#963824` | 主操作 hover |
| `sage-500` | `#6f7d59` | 完成、已归档 |
| `amber-500` | `#b9822e` | 处理中、注意 |
| `red-500` | `#a6382e` | 错误、危险 |

使用规则：

- 朱砂红只用于主按钮、当前导航、重点批注和已收录反馈。
- 不使用蓝紫渐变、玻璃拟态、大面积饱和色。
- 卡片阴影要少，层级优先靠纸面色、边线、排版密度建立。
- 状态色必须带文字，不只靠颜色表达。

## 3. Typography

字体策略：

| Role | Font Stack | Usage |
|---|---|---|
| Display / Page Title | `"Noto Serif SC", "Songti SC", "SimSun", serif` | 品牌、页面标题、稿件标题 |
| UI / Body | `"Noto Sans SC", "PingFang SC", "Microsoft YaHei", sans-serif` | 导航、按钮、表单、正文 |
| Mono / Data | `"JetBrains Mono", "SFMono-Regular", Consolas, monospace` | 数字、编号、时间 |

排版规则：

- 页面标题 34-44px，衬线，行高 1.12。
- 区块标题 20-24px，衬线或 600 黑体。
- 正文编辑区 15-16px，行高 1.75。
- 标签、状态、编号使用 12px，适当增加字重而不是全大写。
- 中文长段落使用 `text-wrap: pretty`，标题使用 `text-wrap: balance`。

## 4. Layout

桌面端使用 264px 左侧档案导航 + 流式主工作区。主工作区最大宽度 1360px。

页面结构：

- 首页：状态总览 + 最近稿件 + 示例对照，像编辑台首页。
- 资产库：左侧素材列表，右侧表达画像和平台规则。
- 创作页：素材选择、平台规则、生成设置三段式。
- 校准页：左侧来源原文，右侧可编辑稿件，底部反馈条。
- 草稿页：表格化列表，强调稿件状态和反馈标签。

避免：

- 不做居中 hero。
- 不做三列等高功能卡。
- 不把每个模块都做成白卡片。
- 不用过多圆角，出版编辑室的边角应更利落。

## 5. Components

### Buttons

- Radius: 6px。
- Min height: 40px。
- Primary: `bg-seal-500 text-paper-0`。
- Secondary: `bg-paper-0 text-ink-800 border border-paper-200`。
- Quiet: 无底色，hover 出现 `paper-50`。
- Active: `translateY(1px)` 或 `scale(0.98)`。

### Panels

- 稿件编辑器使用白纸面、浅边线、轻阴影。
- 档案列表使用紧凑边线，不使用厚卡片。
- 批注和提醒使用左侧色条或顶部小标签，不用大面积彩色底。

### Tags

- 标签使用 4px 圆角。
- 反馈标签选中后使用朱砂红底。
- 平台标签不要模拟平台品牌色，靠文字规则区分。

### Navigation

- 桌面左侧栏像档案索引。
- 当前项使用朱砂左边线或浅红底。
- 移动端底部导航保留，触控目标不少于 44px。

## 6. Motion

交互克制但要有反馈：

- Hover: 背景轻微加深或文字变深。
- Pressed: 轻微下沉。
- Focus: 清晰可见的朱砂色 focus ring。
- 动画只使用 `transform`, `opacity`, `background-color`, `border-color`。
- 时长 160-220ms。

## 7. Content Tone

文案要像编辑软件，不像营销页。

Do:

- “生成并进入校准”
- “已写入下次生成的约束”
- “来源原文”
- “表达画像”
- “禁用表达”

Don't:

- “释放你的创作潜能”
- “一键打造爆款内容”
- “智能赋能创作者”
- “开启下一代内容革命”

## 8. Implementation Notes

- 使用现有 Next.js + Tailwind，不迁移框架。
- Tailwind token 以 `paper / ink / seal` 为主。
- 全局只放字体、背景、文本渲染、少量工具类。
- 新增组件前先复用 `AppShell`, `SectionTitle`, `TextareaBlock`, `IconPill`。
- 所有中文源码必须保持 UTF-8，不允许再次出现乱码。
