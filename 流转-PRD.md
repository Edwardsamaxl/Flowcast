# 流转 - 产品需求文档

## 1. 产品概述

**流转** 是面向中文知识创作者的「人物声纹资产库 + 视频内容多平台流转工具」。

它帮助创作者把长视频、直播回放、课程录屏、口播素材转化成适合多个平台发布的文字内容，并尽量保持创作者本人一贯的观点、语气和表达方式。

核心价值不是“AI 帮你写一篇文案”，而是：

> 让你的视频内容，按你的表达方式，流转成多个平台都能发布的文字版本。

产品需要同时突出两件事：

1. **像你**：AI 学会某个人物的观点、语气、禁用词、写法偏好。
2. **流转**：同一条视频内容可以被拆解、改写，并转成小红书、知乎、X 等平台可发布版本。

## 2. 产品定位

### 2.1 一句话定位

**流转：把视频内容转成像本人写的多平台文字内容。**

### 2.2 用户心智

用户打开流转，不是为了得到一篇泛泛的 AI 文案，而是为了：

- 把长视频、直播、课程录屏拆成小红书、知乎、X 等平台内容。
- 让 AI 调用指定人物的声纹画像，写得更像这个人。
- 管理不同人物的表达资产，知道当前内容是按谁的声纹生成的。
- 在生成后通过对话和反馈继续修改内容。
- 把有价值的视频内容沉淀成可复用的人设、观点和写法规则。

### 2.3 当前 MVP 边界

MVP 阶段只做单用户、单人或少量人物画像管理。产品结构要为未来多人物画像预留空间，但不做复杂团队协作。

MVP 做：

- 本地上传视频。
- 视频转写。
- 生成或更新人物声纹画像。
- 选择人物画像。
- 选择发布平台。
- 将视频内容改写成平台文字内容。
- 在生成结果旁边通过对话进行反馈和改稿。
- 保存历史任务和反馈。

MVP 不做：

- 自动发布。
- 排期日历。
- 团队协作。
- 多账号管理。
- 浏览器插件。
- NotebookLM 式资料问答。
- 通用标题生成器。
- 独立“校准页”。

## 3. 核心用户

第一阶段服务中文知识创作者，尤其是已经拥有视频或直播内容的人。

| 用户类型 | 特征 | 核心痛点 |
|---|---|---|
| 考研、考证、语言类博主 | 有课程、直播、长视频、资料包 | 内容多，但拆成平台内容耗时且容易不像本人 |
| 职场、成长类知识博主 | 持续输出方法论和案例 | AI 能写，但写不出本人的判断力 |
| 咨询、教练型个人 IP | 内容用于获客和建立信任 | 需要专业、自然、不硬广的文字输出 |
| 小团队个人 IP | 创始人是核心表达者，助理负责执行 | 助理能整理，但写不出创始人口吻 |

## 4. 产品信息架构

一级导航只保留四个页面：

1. **首页**
2. **资产库**
3. **拆解内容**
4. **历史**

删除：

- “编辑台”
- 独立“校准页”
- 不明用途的悬浮球

“校准”不再是一个页面，而是生成结果后的编辑、对话、反馈状态。它出现在“拆解内容”和“历史详情”里。

## 5. 关键页面

### 5.1 首页

首页是产品入口，不是工作台。

主文案：

> 让视频内容自然流转。

副文案：

> 建立人物声纹资产库，让 AI 学会你的观点、语气和表达方式。无论是长视频还是直播内容，都能转化成像本人写的多平台文字内容。

核心 CTA：

- 创建声纹资产库
- 拆解一条内容

首页必须展示：

- 产品一句话解释。
- 两个明确入口。
- 一个简短示例：同一段视频如何转成不同平台内容。
- 一个“像本人写”的对比片段。

首页不应该展示：

- 工作台状态。
- 复杂数据卡片。
- 历史任务列表。
- 不明用途的悬浮操作。

### 5.2 资产库

资产库的目标是：管理“当前在记录谁的声纹”。

页面采用左右布局。

左侧：源文件与上传区

- 上传视频。
- 展示已上传过的源文件。
- 展示文件状态：转写中、已转写、已提取画像、失败。
- 点击源文件可以查看转写文本和提取出的观点。
- 这个区域的上传目的主要是“用于建立或更新声纹资产库”。

右侧：人物声纹画像

- 当前人物。
- 人设定位。
- 语气特征。
- 高频观点。
- 常用结构。
- 禁用表达。
- 平台写法规则。

要求：

- 资产库必须让用户明确知道当前画像属于哪个人物。
- 未来支持多人物画像，所以需要有“人物选择 / 人物管理”的位置。
- 平台写法规则必须可编辑。
- 声纹画像结构可以保留当前思路，但视觉上要更像人物档案，而不是普通表单堆叠。

待讨论：

- MVP 是否允许创建多个人物，还是只保留一个默认人物但 UI 预留选择器。
- 上传视频后是否自动写入人物画像，还是需要用户确认。

### 5.3 拆解内容

拆解内容是核心工作流页面。

目标：用户上传或选择一条视频，选择人物画像和目标平台，然后得到可编辑、可反馈的多平台文字内容。

页面必须保持左右布局。

左侧：视频输入与内容解析

- 拖入视频。
- 选择已有源文件。
- 展示视频解析状态。
- 展示转写文本。
- 展示从视频中提取出的核心观点、案例、金句或段落。

右侧：人物、平台、改写结果和对话反馈

上方配置区：

- 人物画像选择。
  - 默认可为“无”。
  - 可选择已创建的人物画像。
  - MVP 阶段可以只有一个人物，但 UI 需要预留下拉选择。
- 发布平台选择。
  - 小红书。
  - 知乎。
  - X。
  - 后续可扩展。
- 是否沉淀进人物声纹资产库。
  - 默认建议关闭或明确提示。
  - 因为用户有时只是临时拆一条视频，不想污染人设库。

下方输出区：

- 改写后的内容。
- 平台版本切换。
- 保存草稿。
- 重新生成。
- 复制。
- 对话式反馈框。

对话反馈框用于：

- 用户直接说“这段太像营销号，把语气压低一点”。
- 用户要求“保留这个观点，但换成知乎长文结构”。
- 用户要求“把标题再克制一点”。
- 用户决定“这条反馈是否写回人物画像”。

设计问题：

- 右侧同时放配置、输出、对话框可能臃肿。
- 需要通过清晰分区解决，而不是拆出独立校准页。
- 可考虑右侧上下两层：上方为配置条，下方为输出与对话双层面板。

### 5.4 历史

历史页记录每一次拆解任务。

列表页展示：

- 源文件标题。
- 使用的人物画像。
- 输出平台。
- 生成状态。
- 是否有反馈。
- 最近更新时间。

详情页展示：

- 源视频信息。
- 源文本 / 转写文本。
- 使用的人物画像。
- 使用的平台规则。
- 生成稿件。
- 反馈记录。
- 对话式改稿记录。

详情页反馈区不应该只是标签。它应该支持：

- 选择反馈标签。
- 输入自然语言反馈。
- 标记反馈是否写回人物画像。
- 查看本次反馈如何影响后续生成。

## 6. 核心功能

### 6.1 人物声纹资产库

声纹资产库不是简单资料库，而是围绕“人物”组织的表达档案。

一个人物画像包括：

- 人物名称。
- 人设定位。
- 领域。
- 语气特征。
- 高频观点。
- 常用案例。
- 常用句式。
- 标题偏好。
- 常用内容结构。
- 禁用表达。
- 平台写法规则。

MVP 阶段可以只支持一个默认人物，但数据结构应支持多个。

### 6.2 视频内容流转

同一条视频内容可以被转成多个平台版本。

平台规则示例：

- 小红书：短段落、强场景、标题直接、轻转化。
- 知乎：结论先行、结构完整、弱营销、强调可信判断。
- X：短句、观点密度高、开头抓人、适合轻互动。

MVP 阶段先支持：

- 小红书。
- 知乎。
- X。

### 6.3 对话式改稿与反馈

生成内容后，用户可以通过两种方式反馈：

1. 标签反馈：
   - 太 AI 味了。
   - 不像我。
   - 太长了。
   - 太营销了。
   - 语气不对。
2. 自然语言反馈：
   - “标题太夸张，压低一点。”
   - “这段不要像课程广告，改成经验分享。”
   - “保留观点，但换成更适合知乎的论证结构。”

反馈需要区分两种作用范围：

- **只影响当前稿件。**
- **写回人物画像，影响后续生成。**

## 7. 用户流程

### 7.1 创建声纹资产库

```text
首页
  -> 创建声纹资产库
  -> 进入资产库
  -> 上传视频
  -> 系统转写视频
  -> 系统提取人物画像
  -> 用户确认和编辑画像
  -> 保存人物声纹资产
```

### 7.2 拆解一条内容

```text
首页
  -> 拆解一条内容
  -> 进入拆解内容
  -> 上传或选择视频
  -> 系统转写并解析内容
  -> 选择人物画像
  -> 选择目标平台
  -> 生成平台文字内容
  -> 用户编辑或对话反馈
  -> 保存到历史
```

### 7.3 从历史继续反馈

```text
历史
  -> 打开某次拆解任务
  -> 查看源文本和生成稿
  -> 继续对话反馈
  -> 更新当前稿件
  -> 可选择写回人物画像
```

## 8. 数据对象

| 对象 | 作用 |
|---|---|
| User | 用户基础信息 |
| Persona | 人物，表示当前为谁记录声纹 |
| VoiceProfile | 人物声纹画像 |
| SourceVideo | 上传的视频源文件 |
| VideoTranscript | 视频转写文本和解析结果 |

| GeneratedDraft | 某个平台的生成稿件 |
| FeedbackMessage | 对话式反馈记录 |

## 9. MVP 必做

- 首页：两个入口，创建资产库和拆解内容。
- 资产库：上传视频、源文件列表、人物画像、平台规则编辑。
- 拆解内容：左右布局，视频输入、转写文本、人物选择、平台选择、生成结果、对话反馈。
- 历史：任务列表和详情页。
- 删除独立校准页。
- 删除不明用途悬浮球。
- 支持单人物画像，但数据结构预留多人物。
- 支持小红书、知乎、X 三个平台。
- 支持反馈标签和自然语言反馈。
- 支持反馈是否写回人物画像。

## 10. 暂不做

- 自动发布。
- 排期。
- 团队协作。
- 付费系统。
- 多账号矩阵管理。
- 浏览器插件。
- 公众号、朋友圈、口播稿输出。
- 重型知识库问答。
- 四维打分系统。

## 11. 当前待讨论问题

1. **首页主文案**
   - 方案 A：让视频内容自然流转。
   - 方案 B：让 AI 写得像你，也让内容流转起来。
   - 方案 C：把一条视频，流转成多个平台都能发的内容。

2. **资产库上传视频的默认行为**
   - 上传后自动更新人物画像。
   - 上传后只解析，用户确认后再写入人物画像。

3. **拆解内容是否默认沉淀进资产库**
   - 默认不沉淀，避免污染人设。
   - 默认沉淀，但生成前明确提示。

4. **右侧输出和对话是否过于臃肿**
   - 方案 A：输出区和对话区上下排列。
   - 方案 B：输出区为主，对话作为右侧抽屉。
   - 方案 C：输出区和对话区使用 tabs。

5. **多人物画像的 MVP 处理**
   - 只做一个默认人物。
   - 允许创建多个，但不做复杂管理。
   - UI 预留下拉选择，数据先写死一个人物。
