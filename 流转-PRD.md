# 流转 - 产品需求文档

## 1. 产品概述

**流转 (Flowcast)** 是面向中文知识创作者的「内容跨平台流转工具」。

它的核心价值是帮助创作者把已经存在的内容资产，在不同平台、不同表达形态之间重新组织和分发：

> 让创作者把一份内容，按自己的表达方式，流转到更多平台。

很多创作者只熟悉一个平台：有人擅长 B 站长视频，有人擅长抖音口播，有人擅长小红书图文，有人擅长公众号长文。流转要解决的问题是：创作者不必重新学习每个平台的表达习惯，也不必从零改写内容，就能把原有内容转成适合多个平台发布的版本，扩大内容影响力。

远期愿景是让内容在视频、音频、图文、文本之间自由流转：

- 视频 -> 图文 / 长文 / 短帖 / 口播稿。
- 音频 -> 图文 / 长文 / 短帖。
- 文本 -> 视频脚本 / 图文草稿 / 平台短帖。
- 图文 -> 视频脚本 / 长文 / 多平台文案。

当前 MVP 先把最成熟、最刚需的一条链路做清楚：**视频 -> 文字**。但前端信息架构和页面设计必须预留其它流转方向，让用户一眼能感知这是一个“内容流转平台”，而不是只支持三个平台的小文案工具。

产品需要同时突出三件事：

1. **流转**：一份内容可以跨平台、跨形态复用。
2. **像本人**：AI 调用创作者画像，保留创作者的观点、语气、禁用词和写法偏好。
3. **可控沉淀**：任何长期画像更新都需要创作者确认，避免污染创作者资产。

## 2. 产品定位

### 2.1 一句话定位

**流转：让创作者把一份内容，流转成多个平台都能发布的版本。**

### 2.2 用户心智

用户打开流转，不是为了得到一篇泛泛的 AI 文案，而是为了：

- 把长视频、直播回放、课程录屏、口播素材拆成多个平台可发布内容。
- 把一个平台的内容转成另一个平台更适合的表达方式。
- 让 AI 调用指定创作者画像，写得更像这个人。
- 管理多个创作者画像，知道当前内容是按谁的表达方式生成的。
- 在生成后通过对话和反馈继续修改内容。
- 把有价值的视频、音频、图文、文本沉淀成可复用的观点、案例、语气和写法规则。

### 2.3 流转路径

MVP 阶段后端优先实现视频转文字，但产品和前端需要展示完整流转版图。

| 流转方向 | MVP 状态 | 前端展示 | 说明 |
|---|---|---|---|
| **视频 -> 文字** | **MVP 可用** | 强展示 | 上传视频，转写解析，生成多平台文字内容 |
| 音频 -> 文字 | 可预留 | 展示入口，可标注稍后接入 | 与视频共享转写管线 |
| 图文 -> 文字 | 后续 | 展示入口，可标注稍后接入 | 从既有图文提炼观点并改写到其它平台 |
| 文本 -> 多平台文字 | 后续 | 展示入口，可标注稍后接入 | 从文章、笔记、提纲流转到多平台 |
| 文本 -> 视频脚本 | 后续 | 概念展示 | 暂不承诺直接生成视频 |
| 视频 -> 视频拆条 | 后续 | 概念展示 | 后续接入剪辑能力 |

### 2.4 当前 MVP 边界

MVP 做：

- 本地上传视频。
- 视频转写与观点提取。
- 生成或更新创作者画像。
- 选择创作者画像。
- 选择发布平台。
- 将视频内容改写成多平台文字内容。
- 在生成结果旁边通过对话进行反馈和改稿。
- 展示画像将如何变化，由创作者确认是否写入。
- 保存历史任务和反馈。

MVP 前端要预留但暂不实现完整能力：

- 音频、图文、文本入口。
- 文字 -> 视频脚本、视频拆条等远期流转方向。
- 更多平台输出规则。

MVP 不做：

- 自动发布。
- 排期日历。
- 团队协作。
- 多账号管理。
- 浏览器插件。
- NotebookLM 式资料问答。
- 独立校准页。
- 视觉风格画像、声音克隆。
- 直接生成可发布视频或图片。

## 3. 核心用户

第一阶段服务中文知识创作者，尤其是已经拥有内容资产但只熟悉部分平台的人。

| 用户类型 | 特征 | 核心痛点 |
|---|---|---|
| B 站 / 视频号长视频创作者 | 有长视频、直播回放、课程内容 | 内容价值高，但迁移到图文和短帖很耗时 |
| 抖音 / 快手口播创作者 | 擅长短视频表达 | 想扩展到小红书、公众号、知乎，但不熟悉文字平台 |
| 小红书 / 公众号图文创作者 | 擅长图文表达 | 想把观点转成视频脚本或其它平台短内容 |
| 知识付费 / 考研 / 考证博主 | 有课程、直播、资料包 | 内容多，但拆解、改写、分发成本高 |
| 小团队个人 IP | 创始人负责表达，助理负责分发 | 助理能整理，但写不出创始人口吻 |

## 4. 产品信息架构

Landing Page 作为产品入口，用户从 Landing Page 直接进入工作流。

应用内一级导航三个页面：

1. **资产库**
2. **流转工作台**
3. **历史**

删除：

- 首页工作台。
- 编辑台。
- 独立校准页。
- 不明用途的悬浮球。

“校准”不再是一个页面，而是生成结果后的编辑、对话、反馈和写回状态。它出现在“流转工作台”和“历史详情”里。

## 5. 关键页面

### 5.1 Landing Page

Landing Page 是产品入口，不是工作台。用户从这里理解“流转”的产品价值，并直接进入核心工作流。

主文案：

> 让视频、音频、图文、文本自由流转。

副文案：

> 把长视频、直播回放、课程录屏、口播、图文和文章，转成适合不同平台发布的内容版本。当前先开放视频转文字链路，并通过创作者画像尽量保留本人的观点、语气和表达方式。

核心 CTA：

- 创建创作者画像
- 拆解一条内容

Landing Page 必须展示：

- 产品一句话解释。
- 两个明确入口。
- “内容上传 -> 转写解析 -> 用户确认 -> 创作者画像沉淀 -> 多平台输出”的主流程。
- 一个简短示例：同一段视频如何转成不同平台内容。
- 一个“像本人写”的对比片段。
- 多平台覆盖感：小红书、抖音、B 站、视频号、公众号、知乎、X、快手、微博、即刻等平台都应进入前端设计视野。
- 多形态流转愿景：视频、音频、图文、文本入口都可展示，但当前可用状态要区分清楚。

Landing Page 不应该展示：

- 工作台状态。
- 历史任务列表。
- 复杂数据仪表盘。
- 不明用途的悬浮操作。

### 5.2 资产库

资产库的目标是：管理“当前在记录谁的创作者画像”。

页面采用左右布局或三栏布局，核心不是“文件仓库”，而是“创作者画像 + 画像素材证据”。

左侧：创作者管理

- 当前创作者。
- 创作者列表。
- 新建创作者。
- 切换创作者。
- 可支持多人物画像，数据库记录，UI 预留下拉或列表选择。

中间：创作者画像

- 创作者名称。
- 人设定位。
- 领域。
- 语气特征。
- 高频观点。
- 常用案例。
- 常用句式。
- 标题偏好。
- 常用内容结构。
- 禁用表达。
- 平台写法规则。

右侧或下方：画像素材与更新建议

- 上传视频。
- 展示已上传过的素材。
- 展示素材状态：待解析、转写中、已解析、待确认写入、已写入画像、已忽略、失败。
- 点击素材可以查看转写文本、提取出的观点、案例、语气和结构。
- 解析后展示“画像将有什么改变”，由创作者决定是否写入。

要求：

- 全部页面文案统一使用“创作者画像”，不再主打“声纹画像”。
- 资产库必须让用户明确知道当前画像属于哪个创作者。
- 平台写法规则必须可编辑。
- 上传视频后不自动覆盖画像，必须先展示变化，再由创作者确认。
- 写入确认必须展示：写入目标创作者、来源素材、将新增/修改/忽略的内容。

### 5.3 流转工作台

流转工作台是核心工作流页面。它不是单纯的上传内容页面，而是一次内容流转任务的完整操作台。

目标：用户上传或选择一份内容，完成转写解析、画像更新确认、多平台输出和反馈改稿。MVP 当前主链路是视频转文字，但页面应预留音频、图文、文本入口。

页面必须突出“流转”的过程：

```text
内容上传
  -> 转写解析
  -> 用户确认
  -> 创作者画像沉淀
  -> 多平台输出
```

左侧：内容输入与解析

- 输入类型切换：视频、音频、图文、文本。
- 视频入口为 MVP 可用。
- 音频、图文、文本入口可展示但标注稍后接入或暂不可用。
- 上传或选择已有素材。
- 展示解析状态。
- 展示转写文本。
- 展示核心观点、案例、金句、段落和可沉淀画像候选。

右侧：流转配置、输出和反馈

顶部配置区：

- 创作者画像选择。
  - 默认可为“无画像”。
  - 可选择已创建创作者画像。
  - 支持多创作者选择。
- 目标平台选择。
  - 小红书。
  - 抖音。
  - B 站。
  - 视频号。
  - 公众号。
  - 知乎。
  - X。
  - 快手。
  - 微博。
  - 即刻。
  - 后续可扩展。
- 是否允许本次内容进入画像候选。
  - 默认可关闭。
  - 解析后展示画像将有什么改变，由创作者决定是否写入。

右侧主体使用工作模式切换，避免信息堆叠：

1. **上传与解析**
   - 上传内容或选择已有素材。
   - 查看转写摘要、转写全文、观点、案例和金句。
   - 查看本次解析得到的画像候选。
2. **画像更新确认**
   - 展示本次内容会如何改变创作者画像。
   - 将变化分成：新增、修改、建议忽略。
   - 用户可以单条接受、单条忽略、手动编辑后接受。
   - 明确显示写入目标创作者和来源素材。
   - 主按钮为“确认写入创作者画像”。
   - 用户也可以选择“不写入，仅用于本次任务”。
3. **稿件**
   - 平台版本切换。
   - 改写后的内容。
   - 保存草稿。
   - 重新生成。
   - 复制。
4. **反馈**
   - 标签反馈。
   - 自然语言反馈。
   - 只影响当前稿件 / 建议写回创作者画像。

对话反馈框用于：

- “这段太像营销号，把语气压低一点。”
- “保留这个观点，但换成知乎长文结构。”
- “这条适合抖音口播，帮我改成更短的脚本。”
- “把标题再克制一点。”
- “这条反馈写回创作者画像，以后都避免这种表达。”

### 5.4 历史

历史页记录每一次拆解任务，用于查找、复盘和继续修改。

列表页必须支持按创作者筛选：

- 全部。
- 每个具体创作者。
- 无画像。

列表页展示：

- 任务标题。
- 源素材标题。
- 输入类型。
- 使用的创作者画像。
- 输出平台。
- 生成状态。
- 是否有反馈。
- 是否已沉淀到画像。
- 最近更新时间。

详情页展示：

- 源素材信息。
- 源文本 / 转写文本。
- 使用的创作者画像。
- 使用的平台规则。
- 生成稿件。
- 反馈记录。
- 对话式改稿记录。
- 写回创作者画像的记录。

详情页反馈区不应该只是标签。它应该支持：

- 选择反馈标签。
- 输入自然语言反馈。
- 标记反馈是否写回创作者画像。
- 查看本次反馈如何影响后续生成。
- 从历史任务继续打开到流转工作台修改。

## 6. 核心功能

### 6.1 创作者画像资产库

创作者画像不是简单资料库，而是围绕“创作者”组织的表达档案。

一个创作者画像包括：

- 创作者名称。
- 人设定位。
- 领域。
- 语气特征。
- 高频观点。
- 常用案例。
- 常用句式。
- 标题偏好。
- 常用内容结构。
- 禁用表达。
- 平台写法规则。

MVP 支持创建多个创作者画像，数据进入数据库记录。前端主要展示一个完整示例，但所有涉及画像的位置都要预留多创作者选择。

### 6.2 多平台内容流转

同一份内容可以被转成多个平台版本。

平台规则示例：

- 小红书：短段落、强场景、标题直接、轻转化。
- 抖音：口播节奏强、开头抓人、适合短视频脚本。
- B 站：结构完整、信息密度高、适合长视频简介和动态文案。
- 视频号：稳重、可信、适合熟人关系和知识传播。
- 公众号：长文结构、标题克制、适合沉淀完整观点。
- 知乎：结论先行、结构完整、弱营销、强调可信判断。
- X：短句、观点密度高、开头抓人、适合轻互动。
- 快手：口语化、真实感强、适合生活化表达。
- 微博：话题感强、短段落、适合传播和讨论。
- 即刻：观点明确、社区感强、适合轻量思考和讨论。

MVP 后端可以先实现部分平台，但前端设计必须体现多平台流转，不要只展示三个平台。

### 6.3 对话式改稿与反馈

生成内容后，用户可以通过两种方式反馈：

1. 标签反馈：
   - 太 AI 味了。
   - 不像我。
   - 太长了。
   - 太营销了。
   - 语气不对。
   - 不适合这个平台。
2. 自然语言反馈：
   - “标题太夸张，压低一点。”
   - “这段不要像课程广告，改成经验分享。”
   - “保留观点，但换成更适合知乎的论证结构。”
   - “这条适合抖音，帮我改成 30 秒口播稿。”

反馈需要区分两种作用范围：

- **只影响当前稿件。**
- **写回创作者画像，影响后续生成。**

## 7. 用户流程

### 7.1 创建创作者画像

```text
Landing Page
  -> 创建创作者画像
  -> 进入资产库
  -> 新建或选择创作者
  -> 上传视频素材
  -> 系统转写解析
  -> 系统生成画像更新建议
  -> 用户确认和编辑
  -> 写入创作者画像
```

### 7.2 拆解一条内容

```text
Landing Page
  -> 拆解一条内容
  -> 进入流转工作台
  -> 选择输入类型
  -> 上传或选择素材
  -> 系统转写并解析内容
  -> 选择创作者画像
  -> 选择目标平台
  -> 生成多平台内容
  -> 用户编辑或对话反馈
  -> 用户确认是否写回画像
  -> 保存到历史
```

### 7.3 从历史继续反馈

```text
历史
  -> 按创作者筛选任务
  -> 打开某次拆解任务
  -> 查看源文本、生成稿和反馈记录
  -> 继续对话反馈
  -> 更新当前稿件
  -> 可选择写回创作者画像
```

## 8. 数据对象

| 对象 | 作用 |
|---|---|
| User | 用户基础信息 |
| Creator / Persona | 创作者，表示当前为谁记录画像 |
| CreatorProfile | 创作者画像 |
| SourceAsset | 上传的源素材，包含视频、音频、图文、文本 |
| Transcript / ParsedContent | 转写文本和解析结果 |
| RewriteTask | 一次拆解和流转任务 |
| GeneratedDraft | 某个平台的生成稿件 |
| FeedbackMessage | 对话式反馈记录 |
| FeedbackTag | 标签式反馈 |

## 9. MVP 必做

- Landing Page：两个入口，创建创作者画像和流转工作台。
- Landing Page：展示完整流转链路和多平台覆盖感。
